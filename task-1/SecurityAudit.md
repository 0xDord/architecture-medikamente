# Аудит мер по обеспечению безопасности данных

## Список проблемных зон

### 1. Хранение данных

#### 1: Небезопасное хранение конфиденциальных данных
- Текущее состояние: Данные хранятся в файлах Excel, JPG, PDF на общем диске без шифрования
- Риски: 
  - Несанкционированный доступ к файлам
  - Отсутствие контроля доступа на уровне файлов
  - Невозможно отследить кто и когда получил доступ к данным

#### 2: Отсутствие резервного копирования с шифрованием
- Текущее состояние: Нет процесса резервного копирования данных
- Риски: Потеря данных, невозможность восстановления

#### 3: Файловый режим работы 1С
- Текущее состояние: «1С:Бухгалтерия» и «1С:Торговля и склад» работают в файловом режиме
- Риски: 
  - Отсутствие централизованного управления доступом
  - Сложность аудита действий пользователей
  - Риск повреждения базы данных

### 2. Обработка данных

#### 1: Ручная обработка данных без аудита
- Текущее состояние: Все операции выполняются вручную, нет журналирования действий
- Риски: 
  - Невозможно отследить кто и когда изменил данные
  - Нарушение требований 152-ФЗ об учете действий с ПДн
- Требования: 152-ФЗ требует ведение журнала действий с ПДн

#### 2: Двойной учёт в Excel и 1С
- Текущее состояние: Кассиры ведут учёт в Excel и 1С параллельно
- Риски: 
  - Расхождения в данных
  - Дублирование конфиденциальных данных
  - Увеличение поверхности атаки

#### 3: Отсутствие контроля целостности данных
- Текущее состояние: Нет механизмов проверки целостности данных
- Риски: 
  - Некорректные изменения данных
  - Отсутствие возможности восстановления после изменений

### 3. Доступ к данным

#### 1: Отсутствие разграничения доступа на уровне данных
- Текущее состояние: Только доменная аутентификация, нет RBAC/ABAC
- Риски: 
  - Сотрудники имеют доступ ко всем данным пациентов
  - Нарушение принципа минимальных привилегий
  - Невозможность реализации требования "один пациент не должен видеть данные другого"
- Требования: 152-ФЗ требует разграничение доступа к ПДн

#### 2: Отсутствие контроля доступа к файлам на общем диске
- Текущее состояние: Файлы доступны всем пользователям домена
- Риски: 
  - Любой сотрудник может получить доступ к медицинским картам
  - Нарушение врачебной тайны

#### 3: Отсутствие сессионного управления
- Текущее состояние: Нет контроля активных сессий пользователей
- Риски: 
  - Использование украденных учетных данных
  - Отсутствие возможности принудительного завершения сессии


### 4. Передача данных

#### 1: Отсутствие контроля потоков данных
- Текущее состояние: Внутренние потоки данных между IT-продуктами не контролируются
- Риски: 
  - Утечка данных через внутренние каналы
  - Отсутствие шифрования при передаче
  - Невозможность отследить маршрут данных

#### 2: Отсутствие шифрования при передаче
- Текущее состояние: Нет шифрования трафика
- Риски: 
  - Перехват данных в сети
  - Man-in-the-middle атаки

#### 3: Интеграция через OLE без контроля
- Текущее состояние: ККМ связаны с 1С через OLE без контроля безопасности
- Риски: 
  - Уязвимости в компонентах OLE
  - Отсутствие валидации данных


### 5. Мониторинг и аудит

#### 1: Отсутствие аудита действий с данными
- Текущее состояние: Нет журналирования действий с конфиденциальными данными
- Риски: 
  - Невозможно расследовать инциденты
  - Нарушение требований 152-ФЗ
  - Отсутствие доказательств при проверках
- Требования: 152-ФЗ требует ведение журнала действий с ПДн

#### 2: Отсутствие мониторинга нетипичных действий
- Текущее состояние: Нет системы обнаружения аномалий
- Риски: 
  - Невозможность обнаружить утечку данных
  - Позднее обнаружение инцидентов безопасности

#### 3: Отсутствие алертинга
- Текущее состояние: Нет системы оповещений о нарушениях безопасности
- Риски: 
  - Позднее реагирование на инциденты
  - Увеличение ущерба от инцидентов
